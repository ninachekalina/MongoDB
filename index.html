Лучший вариант установки MongoDB — Community MSI + Compass.

Установка

MSI-установщик ставит сервер MongoDB. Compass позволяет смотреть коллекции как таблицы или JSON-документы, а также имеет встроенный mongosh прямо в интерфейсе.

Подключение к серверу

Сначала нужно запустить сервер mongod.exe. Чтобы указать, где хранить данные, используют параметр --dbpath. 

mongod.exe --dbpath="C:\data\db"
Место для рисунка: запущенный сервер MongoDB

В этом окне ты запускаешь сам сервер MongoDB — процесс mongod. Пока он работает, база данных доступна. Если закрыть это окно, сервер остановится и все подключения станут невозможны.

Подключение через mongosh

Не закрывая окно с сервером, открой второе окно командной строки и запусти mongosh, чтобы управлять базой данных.

Место для рисунка: окно mongosh

Для обычной работы достаточно двух окон: сервер в одном окне и клиент в другом.

Частые ошибки

  • Не запускай mongod в том же окне, где работаешь с mongosh.
  • Проверь, что mongosh действительно подключился.
  • Если есть ошибка подключения, значит сервер не запущен или слушает другой адрес.

Команды

Проверить подключение и базы

db.getName() show dbs

Посмотреть коллекции

show collections

Создание базы данных

В MongoDB нет команды для создания базы данных в привычном смысле. База данных появляется автоматически, когда впервые сохраняется документ.

Как создать БД в mongosh:

  1. Переключись на нужную базу данных командой use.
  2. Создай коллекцию и добавь первый документ.
  3. После этого база данных будет создана.
use learning_db
